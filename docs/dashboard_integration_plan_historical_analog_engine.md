# Dashboard Integration Plan: Historical Analog Engine

## Objective

Integrate the Sprint 2 Historical Analog Engine into the existing dashboard as a deterministic analyst workflow.

The dashboard should let an analyst enter or select a question, inspect mapped scenario features, review ranked historical analogs, and open supporting evidence.

## Recommended UI Flow

```text
Question Input
      ↓
Mapped Scenario Summary
      ↓
Historical Analog Table
      ↓
Selected Analog Detail
      ↓
Similarity Breakdown + Evidence + Pathways
      ↓
Analyst Notes
```

## Dashboard Components

### 1. Question Input

Initial Sprint 2 dashboard can use preset questions:

- What happens if China launches another Joint Sword exercise?
- What happens if new semiconductor export controls are announced?
- What happens after a major Taiwan election?

Later versions can support free-text entry using the same deterministic question mapper.

### 2. Mapped Scenario Summary

Display:

- scenario family
- actors
- targets
- severity score
- surprise score
- mapping rationale

This makes the question-to-scenario transformation visible to the analyst.

### 3. Historical Analog Table

Columns:

- rank
- event ID
- event name
- date
- family
- similarity score
- confidence
- evidence count

Rows should be clickable and drive the detail panels.

### 4. Similarity Breakdown Panel

For the selected analog, display component scores:

- event-family match
- actor overlap
- target overlap
- severity proximity
- surprise proximity
- temporal-context fit
- evidence coverage

This panel is the main transparency feature.

### 5. Evidence Drawer

Display:

- event source URL
- source packet path
- linked news evidence
- surprise rationale
- coding notes
- historical CAR context where available

CAR values must be labeled as historical descriptive context only.

### 6. Observed Historical Pathways Panel

Display pathway bullets generated by the prototype:

- later same-family records
- near-term subsequent coded events
- repetition-level notes
- historical context availability

Pathway text should avoid future-tense prediction.

### 7. Analyst Notes Panel

Always show:

```text
Historical analog retrieval only. This output does not forecast future outcomes, estimate future returns, or provide investment recommendations.
```

Also show evidence-quality warnings when linked news rows are placeholders.

## Data Contract

The dashboard can consume:

```text
results/historical_analog_engine/demo_results.json
```

Recommended dashboard state:

| State Field | Source |
|---|---|
| selected question | `questions[].question` |
| mapped scenario | `questions[].mapped_scenario` |
| selected analog | `questions[].historical_analogs[]` |
| score breakdown | `selected_analog.score_components` |
| evidence items | `selected_analog.evidence_items` |
| pathways | `selected_analog.observed_historical_pathways` |
| analyst notes | `questions[].analyst_notes` |

## Integration Phases

### Phase 1: Static Demo Integration

- Run `scripts/historical_analog_engine.py`.
- Load `demo_results.json` in the dashboard.
- Add a scenario selector for the three demo questions.
- Render analog table and detail panels.

### Phase 2: Free-Text Local Mapping

- Add a question input box.
- Call the same deterministic mapping rules in the frontend or through a local generated JSON file.
- Display mapping confidence and fallback behavior.

### Phase 3: Analyst Review Workflow

- Add analyst notes field.
- Add retrieval quality labels.
- Add source verification status.
- Add export-to-Markdown for analyst briefs.

### Phase 4: Scalable Product Layer

- Split taxonomy mapping into a standalone config file.
- Generate feature tables during data validation.
- Add dashboard filters by taxonomy, actor, target, confidence, and evidence quality.
- Preserve deterministic scoring as the default retrieval mode.

## Non-Goals

- No LLM API calls.
- No forecasting.
- No investment recommendations.
- No automated causal claims.
- No portfolio allocation or trading guidance.
